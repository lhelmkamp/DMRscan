Laura Helmkamp and Robert Podolsky

2015-09-28

1: Introduction

DNA methylation is a crucial and commonly studied epigenetic mechanism for the regulation of gene expression. Differentially methylated regions (DMRs) have been implicated in a number of diseases, as well as in a variety of developmental and aging processes.

The DMRscan package allows users to accurately find DMRs using the scan statistic. DMRscan constitues a convenient pipeline linking the sitewise likelihood-based differential methylation statistics calculated by the package methylSig, the free scan statistic software SaTScan (via RSaTScan), and the plotting capabilities of Gviz [1,2,3,4].

This document is a step-by-step user guide for the DMRscan package.

2: Installation

2a: Installing SaTScan

DMRscan requires the stand-alone software SaTScan. This can be easily downloaded from http://www.satscan.org/download.html for Windows, Linux, or Mac OS X.

2b: Installing DMRscan

Install DMRscan with the devtools R package:

library(devtools) 
install_github('lhelmkamp/DMRscan')

DMRscan requires that input data be formatted as methylSigData object, as is output from the methylSigReadData() function in the methylSig package. If your data is not in this format, please see the Appendix.

4: Finding DMRs with DMRscan

With data in a methylSigData object, the function DMRscan() can be used to find DMRs. DMRscan() performs the following steps:

  1. Obtain sitewise likelihood ratio statistics
  2. Normalize the sitewise statistics using quantiles
  3. Write the files needed by SatScan to a local directory
  4. Run SatScan and view the results

We start with a simple call to DMRscan() for the chromosome 18 data:

time0.a<-proc.time()
result.a<-DMRscan(CMML_chr18, xvalues = "Index")
time1.a<-proc.time()

Here, xvalues = "Index" indicates that in the scan statistic, the data points will be treated as equally spaced rather than using the position of the site on the chromosome. We include timing code to allow the user to estimate computational time:

(time1.a-time0.a)/60
##         user       system      elapsed 
##  2.656666667  0.004166667 13.398833333

On our computer, where this test of chromosome 18 takes about 13 minutes, analyzing the entire genome-wide dataset for the CMML data takes about 16 hours.

Note that the result is also automatically output to result; we output the result as a precaution due to the long computational time of DMRscan() on large datasets. However, we recommend returning output to a more desciptive name as well, as result will be overwritten by subsequent calls to DMRscan().

If multiple chromosomes are present in the input meth object, DMRscan() will automatically analyze each chromsome separately. For human data, the user can also split each chromsome at the centromere by setting splitcentromere = TRUE and by specifying build from "hg18", "hg19", or "hg38". This will speed up the analysis somewhat, and may be necesary for large datasets if system memory prevents the entire chromosome from being analyzed simultaneously.

time0.b<-proc.time()
result.b<-DMRscan(CMML_chr18, xvalues = "Index", splitcentromere = TRUE, build = "hg18")
time1.b<-proc.time()

5: DMR visualization

We can now visualize the significant regions with the function plotSatScanresult. The results will be output to a pdf in mydir, if this was specified, or in the current working directory. By default, all the significant results returned by plotSatScanresult will be plotted; the options plottopn and plotpval can be used to restrict plotting to the top few results by p-value or to results with a specified p-value more conservative than 0.05, respectively. Plots can also be output directly from the call to plotSatScanresult by specifying plotresult=TRUE.

Plotting uses Gviz to plot the DMR along with an ideogram showing the position of the DMR on the chromosome, the log-likelihood values of the data, and nearby genes, promoter regions, and CpG islands.

plotSatScanresult(result.b)

Appendix: Creating a methylSigData object

A methylSigData object can be created from CpG methylation score files using the methylSigReadData() function in the methylSig package. For a full description of the package and function options, see http://sartorlab.ccmb.med.umich.edu/software. We will include a brief example here for simplicity.

In this example, we have previously downloaded all of the files from GSE into the working directory.

First we check the format of one of the text files. If your data is not currently in subject-wise text files, data can be rearranged and output to text files that have the correct format. From the MethylSig vignette:

The CpG methylation score file must contain at least seven columns. Among these, second to seventh column must be, in order, chromosome, base, strand, coverage, percentage of Cs and percentage of Ts. Column names are not important. Strand format is F/R or +/-, where F/+ represents forward and R/- represents reverse strand.

fileList = dir()
subj1data_chr18<-read.table(fileList[1], header=TRUE)
